Component TouchScreen
Touch Screen
Component Level: High
Category: CPU External Devices-Display
Methods:
(Methods are user-callable functions/subroutines intended for the component functions control. Please see the Embedded Components page for more information.)
  • isCalibrated - Returns TRUE if the touch screen is already calibrated

      ANSIC prototype: bool isCalibrated(void)
    • Return value:bool - TRUE if the touch screen is calibrated, FALSE otherwise.

  • GetPositionRaw - Decides if the screen has been touched or not, and returns the raw touch position

      ANSIC prototype: bool GetPositionRaw(ComponentName_TouchSensorValue *RawTouchPositionX, ComponentName_TouchSensorValue *RawTouchPositionY)
    • RawTouchPositionX: Pointer to ComponentName_TouchSensorValue - Pointer to position variable for X value
    • RawTouchPositionY: Pointer to ComponentName_TouchSensorValue - Pointer to position variable for Y value
    • Return value:bool - TRUE if display has been touched, FALSE otherwise

  • GetPosition - Decides if the screen has been touched or not, and returns the touch position

      ANSIC prototype: bool GetPosition(ComponentName_PixelDim *TouchPositionX, ComponentName_PixelDim *TouchPositionY)
    • TouchPositionX: Pointer to ComponentName_PixelDim - Pointer to position variable for X coordinate
    • TouchPositionY: Pointer to ComponentName_PixelDim - Pointer to position variable for Y coordinate
    • Return value:bool - TRUE if display has been touched, FALSE otherwise

  • DrawCalibrationPoint - Draws a calibration point (crosshair)

      ANSIC prototype: void DrawCalibrationPoint(ComponentName_PixelDim x, ComponentName_PixelDim y, ComponentName_PixelColor color)
    • x:ComponentName_PixelDim - x coordinate
    • y:ComponentName_PixelDim - y coordinate
    • color:ComponentName_PixelColor - color to be used for the crosshair

  • Calibrate - Calibrates the screen. It asks the user to touch the screen to calculate the calibration data.

      ANSIC prototype: void Calibrate(ComponentName_PixelColor fgColor, ComponentName_PixelColor bgColor)
    • fgColor:ComponentName_PixelColor - color to be used for the display foreground
    • bgColor:ComponentName_PixelColor - color to be used for the background color

  • Scan - Scans the touchscreen and creates events. Needs to be called periodically.

      ANSIC prototype: void Scan(void)

  • GetCalibrationData - Returns the calibration data (e.g. to the caller who will store it on a permanent storage).

      ANSIC prototype: void GetCalibrationData(byte **calibDataP, byte *nofBytes)
    • calibData: Pointer to void* - Pointer to pointer to the calibration data. Here the method returns the pointer to the data.
    • nofBytes: Pointer to byte - Pointer to number of bytes. Here the method returns the size of the calibration data.

  • SetCalibrationData - Returns the calibration data (e.g. to the caller who will store it on a permanent storage).

      ANSIC prototype: void SetCalibrationData(byte *calibData, byte nofBytes)
    • calibData: Pointer to void* - Pointer to pointer to the calibration data
    • nofBytes:byte - Number of bytes.

  • Init - Initializes the driver and the Touch Screen Driver

      ANSIC prototype: void Init(void)


Note: Some methods can be implemented as macros.
