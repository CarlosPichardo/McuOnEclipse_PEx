Component MPR08x
MPR083 and MPR084 capacitative touch sensor
Component Level: Low
Category: CPU External Devices-Keypad/Keyboard
Settings for the MPR083 demo board (DEMOMPR083) with HCS08GT16
  • Device: MPR083
  • ATTN: on PTC0
  • IRQ: on IRQ
  • Sensitivity Treshold: around 8
  • Button LEDs:
    • LED0 (for button 0): PTD0
    • LED1 (for button 1): PTD1
    • LED2 (for button 2): PTD3
    • LED3 (for button 3): PTD4
    • LED4 (for button 4): PTC5
    • LED5 (for button 5): PTC6
    • LED6 (for button 6): PTE2
    • LED7 (for button 7): PTE3
  • Freescale Logo:
    • ..1.........PTB0
    • ...2........PTA7
    • ....3.......PTA6
    • ...4........PAT5
    • ..5.6......PTA3 PTA4
    • ...7........PTA2
    • ..8.........PTA1
Settings for 'Can Your Badge Board Do This' (MCF51JM128 with MPR084)
  • Device: MPR084
  • ATTN: on PTC6_RxCAN
  • IRQ: on IRQ_TPMCLK
Note that depending on your processor speed you need to increase the I2C clock to a higher value. If your processor gets again an interrupt from the device while communicating to the I2C, then this may lock the device. Additionally, if your CPU clock speed is too low, then you may not receive the interrupts from the device any more if pads are touched too fast: try to increase the CPU clock speed.

Known areas for improvements
  • The driver is not using burst mode (e.g. in device initialization)
