Component MPR08x
MPR083 and MPR084 capacitative touch sensor
Component Level: Low
Category: CPU External Devices-Keypad/Keyboard
Properties:
(Properties are parameters of the component. Please see the Embedded Components page for more information.)
  • Component name - Name of the component.
  • Hardware - Configures the device hardware.
    • Device Type - Specifies the device used

      There are 2 options:
      • MPR083: Device is the 8 pad rotary sensor MPR083
      • MPR084: Device is the 8 pad single key sensor MPR084.

    • ATTN Interface - ATTN (Attention) pin interface to the device. Needed to wake up the device out of run2 mode.
    • I2C - IIC (or I2C) is used as communication to the device.
      • I2C Interface - Interface to IIC communication channel.
      • Stop I2C after communication - If IC2 shall be stoped after communication to the devide (and started before). This can increase battery life.
    • AD0 Pin is low - If the AD0 pin is put on ground/low or not. This defines the I2C address of the device.
    • Interrupts - If the device shall support interrupts. The bean creates the macro INTERRUPTS_ENABLED in the driver header file which can be used in the application.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • IRQ Interface - Specifies the interface to the interrupt signal generated by the sensor.
  • Initialization - This group contains values to initialize the device at bean initialization time.
    • Run Mode - Defines the initial run mode.

      There are 4 options:
      • Run1: When in Run1 mode the sensor controller will run continuously.
      • Run2: When in Run2 mode the sensor controller will continue to scan the electrodes but a low power state will be enabled between each cycle. Because of this, any I2C communication that occurs, may or may not respond while the sensor is in this mode.
      • Stop1: When in Stop1 mode the sensor controller will not scan the electrodes. While capacitance sensing is disabled I2C communications will still be accepted and the sensor controller will maintain instantaneous response to all register requests. This is the only mode in which register values can be set.
      • Stop2: When in Stop2 mode the sensor controller will not scan the electrodes or accept I2C communication. The MPR083 is off during this mode.

    • Interrupt Rate - The Interrupt Rate Field selects the amount to multiply the MTP by to determine the minimum delay between sequential Interrupts.
    • Touch Acquistion Sample Factor - Touch Acquisition Sample Period (Touch Acquisition Sample Period Register) - The Touch Acquisition Sample Period Field selects or reports the multiplication factor that is used to determine how often electrodes are scanned. The resulting factor must be in the range 1 to 32.
    • Master Tick Period - The Master Tick Period Register is used to set the master tick of this system. All parts of the system are synchronized to this counter. This register is overriden in all modes except for Run1. When not in Run1 mode, the value of this register is ignored and 8 ms is used for the primary clock.
    • Minimum Interrupt Period - Minimal time between two interrupts
    • MPR083 - Settings for the MPR083
      The following items are available only if the group is enabled (the value is "Enabled"):
      • Sensitivity Threshold - Sensitivity Threshold (Sensitivity Threshold Register) - The Sensitivity Threshold selects or reports the sensitivity setting of the Sensor Controller. The resulting value must be in the range 1 to 64 units. 1 is the maximum sensitivity.
    • MPR084 - Settings for the MPR084

      • Maximum number of touched positions - This adjusts the number of keys that can be concurrently reported as touched.
      • Electrodes - Inside this group individual electrodes can be enabled/disabled. Additionally the sensitivity level can be adjusted.

    • Sensor - Configures the sensor (sensor pad configuration register)
      • Enable - The Sensor Enable bit enables or disables the touch sensor. When disabled, no touches are detected.
      • Stuck Key Timeout - The Stuck Key Timeout field selects or reports the multiplication factor that is used to determine how often electrodes are calibrated while a touch is being detected. 0 turns off Stuck Key Detection.
      • Auto Calibration - Auto Calibration Enable - The Auto Calibration Enable bit enables or disables the auto calibration function.
    • Sounder - Settings for the sounder/buzzer.
      • Enable - Sounder Enable - The Sounder Enable bit enables or disables the sounder output.
      • Frequency - Frequency - The Frequency bit controls the frequency of the driven output.

        There are 2 options:
        • 1 kHz: Sounder Frequency is 1 kHz
        • 2 KHz: Sounder Frequency is 2 kHz

      • Click Period - Click Period - The Click Period bit controls the length of the sounder click.

        There are 2 options:
        • 10 ms: 10 ms Sounder Click Period
        • 20 ms: 20 ms Sounder Click Period

    • FIFO - Configuration of the FIFO.
      • FIFO mode - If touches shall be read from FIFO. Otherwise the instantaneous way (reading sensor status register) is used.
      • Detect Touch - FIFO Touch Buffer Enable - The FIFO Touch Buffer Enable bit determines whether or not data is logged in the FIFO any time a button is pressed.
      • Detect Release - FIFO Release Buffer Enable - The FIFO Release Buffer Enable bit determines whether or not data is logged in the FIFO when the button transitions from a touched to untouched state.
    • Low Power - Defines the low power configuration of the device.
