Component PCA9554BitIO
NXP PCA9554 and PCA9554A I2C I/O Expander (BitIO)
Component Level: Low
Category: SW
Methods:
(Methods are user-callable functions/subroutines intended for the component functions control. Please see the Embedded Components page for more information.)
  • GetVal - Returns the input/output value. If the direction is input then the input value of the pin is read and returned. If the direction is output then the last written value is returned.

      ANSIC prototype: bool GetVal(void)
    • Return value:bool - Error code

  • SetVal - Sets (to one) the output value. It is equivalent to the PutVal(TRUE). This method is available only if the direction = output.

      ANSIC prototype: void SetVal(void)

  • ClrVal - Clears (set to zero) the output value. It is equivalent to the PutVal(FALSE). This method is available only if the direction = output.

      ANSIC prototype: void ClrVal(void)

  • NegVal - Negates (inverts) the output value. It is equivalent to the PutVal(!GetVal()). This method is available only if the direction = output.

      ANSIC prototype: void NegVal(void)

  • PutVal - The specified output value is set. If the direction is output, it writes the value to the pin. (Method is available only if the direction = output).

      ANSIC prototype: void PutVal(bool Val)
    • Val:bool - Output value. Possible values:
      false - logical "0" (Low level)
      true - logical "1" (High level)

  • InitPCA9554 - initialize the PCA9554BitIO, it sets the direction

      ANSIC prototype: void InitPCA9554(void)


Note: Some methods can be implemented as macros.
