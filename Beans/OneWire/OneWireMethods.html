Component OneWire
Component for the 1-Wire Protocol
Component Level: High
Category: SW-Communication
Methods:
(Methods are user-callable functions/subroutines intended for the component functions control. Please see the Embedded Components page for more information.)
  • CalcCRC - Calculates the CRC over a number of bytes

      ANSIC prototype: uint8_t CalcCRC(uint8_t *data, uint8_t dataSize)
    • data: Pointer to uint8_t - Pointer to data
    • dataSize:uint8_t - number of data bytes
    • Return value:uint8_t - calculated CRC

  • SendByte - Sends a single byte

      ANSIC prototype: uint8_t SendByte(uint8_t data)
    • data:uint8_t - Variable to save the byte.
    • Return value:uint8_t - error code

  • Receive - Programs a read operation after the master send all in output buffer. Don't use a SendReset while the data is coming.

      ANSIC prototype: uint8_t Receive(uint8_t counter)
    • counter:uint8_t - Number of bytes to receive from slave
    • Return value:uint8_t - error code

  • SendReset - Sends a reset to the bus

      ANSIC prototype: uint8_t SendReset(void)
    • Return value:uint8_t - error code

  • Count - Returns the number of elements stored on input buffer that are ready to read.

      ANSIC prototype: uint8_t Count(void)
    • Return value:uint8_t - number of elements

  • Waitms - Programs a pause between instruccions.

      ANSIC prototype: uint8_t Waitms(uint8_t key, uint8_t time_ms)
    • key:uint8_t - Key to identify the source of the event.
    • time_ms:uint8_t - Value of time to wait.
    • Return value:uint8_t - error code

  • ProgramEvent - Used to add a program event

      ANSIC prototype: uint8_t ProgramEvent(uint8_t key)
    • key:uint8_t - Key to identificate the event, applies only if OnProgramEvent is enabled. (Valid range 0 - 31)
    • Return value:uint8_t - error code

  • SendBytes - Sends multiple bytes

      ANSIC prototype: uint8_t SendBytes(uint8_t *data, uint8_t count)
    • data: Pointer to uint8_t - Array of bytes to add to output stream.
    • count:uint8_t - Number of bytes to add to output stream. (Valid range 0 - 31)
    • Return value:uint8_t - error code

  • GetBytes - Gets multiple bytes from the bus

      ANSIC prototype: uint8_t GetBytes(uint8_t *data, uint8_t count)
    • data: Pointer to uint8_t - Pointer to where to store the data
    • count:uint8_t - Number of bytes
    • Return value:uint8_t - error code

  • GetByte - Get a single byte from the bus

      ANSIC prototype: uint8_t GetByte(uint8_t *data)
    • data: Pointer to uint8_t - Pointer to were to store the data
    • Return value:uint8_t - error code

  • GetError - Returns the error

      ANSIC prototype: void GetError(void)

  • isBusy - Returns TRUE if the bus is busy, FALSE otherwise

      ANSIC prototype: bool isBusy(void)
    • Return value:bool - TRUE if device is busy

  • strcatRomCode - Appends the ROM code to a string.

      ANSIC prototype: uint8_t strcatRomCode(uint8_t *buf, size_t bufSize, uint8_t *romCode)
    • buf: Pointer to uint8_t - Pointer to zero terminated buffer
    • bufSize:byte - size of buffer
    • romCode: Pointer to uint8_t - Pointer to 8 bytes of ROM Code
    • Return value:uint8_t - error code

  • ReadRomCode - Read the ROM code. Only works with one device on the bus.

      ANSIC prototype: uint8_t ReadRomCode(uint8_t *romCodeBuffer)
    • romCodeBuffer: Pointer to uint8_t - Pointer to a buffer with 8 bytes where the ROM code gets stored
    • Return value:uint8_t - Error code

  • ResetSearch - Reset the search state

      ANSIC prototype: void ResetSearch(void)

  • TargetSearch - no hint

      ANSIC prototype: void TargetSearch(uint8_t familyCode)
    • familyCode:uint8_t - family code to restrict search for

  • Search - no hint

      ANSIC prototype: bool Search(uint8_t *newAddr, bool search_mode)
    • newAddr: Pointer to uint8_t - Pointer to 8 bytes of data where to store the new address
    • search_mode:bool - no hint
    • Return value:bool - TRUE if new device has been found, FALSE otherwise.

  • ParseCommand - Shell Command Line parser. Method is only available if Shell is enabled in the component properties.

      ANSIC prototype: uint8_t ParseCommand(const unsigned char* cmd, bool *handled, const %@Shell@'ModuleName'%.StdIOType *io)
    • cmd:byte - command string
    • handled: Pointer to bool - Pointer to variable which tells if the command has been handled or not
    • io:byte - Pointer to I/O structure
    • Return value:uint8_t - Error code

  • Deinit - Driver de-initialization

      ANSIC prototype: void %'ModuleName'%.Init(void) Deinit(void)

  • Init - Initializes this device.

      ANSIC prototype: void %'ModuleName'%.Init(void) Init(void)


Note: Some methods can be implemented as macros.
