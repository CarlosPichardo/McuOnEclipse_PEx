Bean I2CSpy
I2C Bus Spy
Component Level: High
Category: SW-User Components
Methods:
(Methods are user-callable functions/subroutines intended for the component functions control. Please see the Embedded Components page for more information.)
  • Init - Initialization method

      ANSIC prototype: byte Init(void)
    • Return value:byte - Error code

  • Deinit - Deinitialization method

      ANSIC prototype: void Deinit(void)

  • SetDeviceAddress - Sets the I2C Device address to be used

      ANSIC prototype: byte SetDeviceAddress(byte addr)
    • addr:byte - I2C 7bit address
    • Return value:byte - Error code

  • GetDeviceAddress - Returns the current I2C Device address used

      ANSIC prototype: byte GetDeviceAddress(void)
    • Return value:byte - 7bit Device Address

  • SetAddressSize - Specifies the address size of the I2C device memory

      ANSIC prototype: byte SetAddressSize(byte size)
    • size:byte - Either 1 (8bit), 2 (16bit), 3 (24bit), or 4 (32bit)
    • Return value:byte - Error code

  • SetBytesPerLine - Sets the number of bytes per line for the Dump shell command

      ANSIC prototype: byte SetBytesPerLine(byte nofBytesPerLine)
    • nofBytesPerLine:byte - Number of bytes per line
    • Return value:byte - Error code

  • ReadRegData - Reads data starting from a register address from the current device

      ANSIC prototype: byte ReadRegData(dword addr, byte *data, word dataSize)
    • addr:dword - Device address
    • data: Pointer to byte - Pointer to data buffer where read data is stored
    • dataSize:word - size of data buffer in bytes
    • Return value:byte - Error code

  • WriteRegData - Writes data starting from a register address to the current device

      ANSIC prototype: byte WriteRegData(dword addr, byte *data, word dataSize)
    • addr:dword - Address in device
    • data: Pointer to byte - Pointer to data to write
    • dataSize:word - Size of data in bytes
    • Return value:byte - Error code

  • ParseCommand - Shell Command Line parser

      ANSIC prototype: byte ParseCommand(const unsigned char *cmd, bool *handled, const %@Shell@'ModuleName'%.StdIOType *io)
    • cmd: Pointer to byte - Pointer to command string
    • handled: Pointer to bool - Pointer to variable which tells if the command has been handled or not
    • io:byte - Pointer to I/O structure
    • Return value:byte - Error code


Note: Some methods can be implemented as macros.
