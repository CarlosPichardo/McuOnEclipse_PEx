User Interface for different displays
Component Level: High
Category: CPU External Devices-Display
Application Notes:
(Examples, notes and hints for the component usage.)
The userinterface is the main part of the LCD-Widget Library. The userinterface contains all common methods for the widgets.
Every widget-component needs the userinterface. It is not possible to program a widget whitout the userinterface.
The screen-widget is now part of the userinterface, so the user can update the whole screen via the method: UpdateScreen.
The selection of an element can be enabled or disabled in the userinterface, but every element defines the look of a selected element itself.
  #include "UI1.h"
  #include "UIWIN1.h" /* from WindowWidget component */
  #include "UIB1.h" /* from ButtonWidget component */

  /* Data structure */
  UI1_Screen screen; /* screen where all widgets are painted */
  UIWIN1_WindowWidget window; /* from WindowWidget component */

  /* constructor */
  UI1_CreateScreen(&screen, UI1_COLOR_GREEN); /* creates a green screen */

  /* window contructor */
  (void)UIWIN1_CreateWindow(&screen, &window, UI1_COLOR_WHITE, 0, 0, UI1_GetWidth(), UI1_GetHeight()); /* creates a window, covering the whole screen */

  /* add here any other widget constructors, i. e. a button*/
  (void)UIB1_CreateButton(&App.u.myAppW.window, &App.u.myAppW.uiButton, 5, 140, 0, 0, "Widget Button", FONT_18, UI1_COLOR_BLUE);

  /* write your own callback methods */
  ...

  /* after creation update whole screen */
  UI1_UpdateScreen(&screen, TRUE);
